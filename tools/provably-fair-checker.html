How Provably Fair Verification Works

Before you place a bet, the casino commits to a server seed by showing you a SHA-256 hash of that seed. This hash is mathematically impossible to reverse, so the casino cannot change the seed after you see the hash. Once your bet is placed using your client seed and nonce, the outcome is determined. After the round ends, the casino reveals the original server seed.

Your verification has two steps: first, hash the revealed server seed and confirm it matches the hash shown before your bet. Second, recalculate the game outcome using all three inputs (server seed, client seed, nonce) and confirm it matches what the game showed.

Step-by-Step Verification Guide

  1. Before betting: note the server seed hash shown in the game's fairness panel.
  2. Optionally change your client seed to one you chose yourself.
  3. Place your bet. The nonce increments with each bet.
  4. After the round, click "Reveal" in the fairness panel to see the server seed.
  5. Enter all values above and click Verify to confirm the hash matches.
  6. If the hashes match, the casino could not have changed the outcome after you bet.

HMAC-SHA256 vs SHA-256

Different platforms use slightly different implementations. Stake.com uses HMAC-SHA256 with the server seed as the key and the client seed combined with the nonce as the message. BC.Game uses a similar approach. The tool above calculates both the SHA-256 hash of the server seed (for hash verification) and the HMAC-SHA256 of the combined inputs (for outcome calculation).

For full outcome verification, you also need to know the specific game's outcome algorithm, which converts the hash result into a game outcome (e.g., a dice roll number or crash point). Each game type has its own conversion method, documented in the platform's fairness documentation.

Frequently Asked Questions

Does this tool share my seeds with anyone?

No. This tool runs entirely in your browser using the Web Crypto API. No data is sent to CryptoBets.com or any other server. You can verify this by inspecting the page source.

What is a nonce in provably fair gambling?

A nonce is a sequential counter that increments with each bet. It ensures that even with the same server and client seeds, every bet produces a different result. Starting nonce is typically 0 or 1 and increases by 1 per bet.

Can I change my client seed?

Yes. Most provably fair platforms let you change your client seed at any time before placing a bet. Using a custom seed you chose yourself means the casino had no control over that input.

What if the hashes do not match?

If the revealed server seed does not hash to what was shown before your bet, the casino changed its seed after you bet. This would be a provably fair violation. Document the mismatch and contact the platform's support immediately.

Do slot machines use provably fair?

No. Third-party slot machines (Pragmatic Play, NetEnt, etc.) use certified RNG audited by independent labs, not provably fair algorithms. Provably fair applies only to casino-original games.
